# Documentación de Misión 1-99

Este índice separa las fuentes que gobiernan el trabajo de los materiales que ayudan a aprenderlo. Ante una contradicción sobre el estado implementado, se aplica la precedencia definida en el [fundamento técnico](./technical-foundation.md).

## Ruta de lectura recomendada

1. [Resumen ejecutivo](./executive-summary.md): propósito, actores, alcance y criterios de éxito.
2. [Fundamento técnico](./technical-foundation.md): arquitectura actual, invariantes, evidencia y brechas objetivo.
3. [Hoja de ruta](./development-roadmap.md): prioridad, estado y dependencias de los slices.
4. [Proceso de desarrollo](./development-process.md): secuencia normativa desde el vertical slice hasta entrega, release y `archive`.
5. [Glosario](./glossary.md): definiciones canónicas para interpretar los documentos anteriores.
6. [De una idea a una entrega verificable](./from-idea-to-delivery.md): explicación educativa y ejemplo completo del proceso.

## Fuentes normativas

| Fuente                                            | Gobierna                                                                          |
| ------------------------------------------------- | --------------------------------------------------------------------------------- |
| [Resumen ejecutivo](./executive-summary.md)       | Propósito, actores, alcance y no objetivos del MVP.                               |
| [Fundamento técnico](./technical-foundation.md)   | Arquitectura vigente, invariantes técnicas, estado comprobado y brechas objetivo. |
| [Hoja de ruta](./development-roadmap.md)          | Prioridad, estado y dependencias de capacidades y slices.                         |
| [Proceso de desarrollo](./development-process.md) | Secuencia SDD, evidencia, Definition of Ready/Done, rollback y review budget.     |
| [Glosario](./glossary.md)                         | Significado canónico de términos compartidos.                                     |
| [`AGENTS.md`](../AGENTS.md)                       | Reglas de contribución, comandos y límites del repositorio.                       |

## Material educativo

[De una idea a una entrega verificable](./from-idea-to-delivery.md) enseña el razonamiento detrás del proceso mediante distinciones, heurísticas y un ejemplo. No crea requisitos ni reemplaza las fuentes normativas.
